# picoCTF - Dont-use-client-side

## 問題概要
- ジャンル：Web Exploitation
- 難易度：Easy
- 内容：ログイン画面のクライアント側処理を解析する問題

## 取り組み方
- Webページを開き、開発者ツールでHTML/JavaScriptを確認
- `<script>` 内に認証ロジックがあると考えた

## 解法
- JavaScript内の `verify()` 関数を確認
- 入力値を `substring()` で分割し、固定文字列と比較していることを確認
- 各文字列を順に連結してflagを復元

## 学んだこと
- クライアントサイドでの認証処理は攻撃者から完全に可視である
- 認証や秘密情報の検証は必ずサーバーサイドで行うべき

## 次に活かす視点
- Web問題では、まずクライアント側で重要な判定をしていないか確認する
